## 1. Product definition (locked scope)

**What we are building:** A local, secure bridge that lets AI chat agents inside VS Code (Copilot Chat, Claude Desktop, Cursor, Cline, Continue) inspect and modify a running Blender scene through MCP tool calls.

**Three shipped components:**
- `blender_addon/` — Blender add-on. Runs a WebSocket server on `127.0.0.1`, executes `bpy` ops on the main thread.
- `mcp_server/` — Python MCP server. Exposes typed tools to AI clients, validates, applies policy, forwards to add-on.
- `vscode_extension/` — Optional TypeScript extension. Approval webview, viewport preview, output channel.

**Out of scope (MVP):** cloud hosting, multi-user, non-loopback transports, telemetry, marketplace publish, GPU-cluster rendering, animation timeline editor UI.

## 2. Locked technical decisions (do not relitigate)

| Topic | Decision |
|---|---|
| Transport AI ↔ server | **MCP** (stdio JSON-RPC, official `mcp` Python SDK ≥ 1.2) |
| Transport server ↔ Blender | **WebSocket** on `ws://127.0.0.1:9876` (configurable) |
| Auth | 32-byte URL-safe token, generated by add-on, stored in OS keyring on server side (`keyring` lib) |
| Languages | Add-on: Python 3.11 (Blender 4.2 LTS bundled). Server: Python 3.11. Extension: TypeScript 5.x. |
| Schema source of truth | JSON Schema in `shared/schemas/` → generated to Pydantic + `.d.ts` |
| Concurrency in Blender | `asyncio` WS in daemon thread → `queue.Queue` → `bpy.app.timers` pump every 10 ms |
| Undo | `bpy.ops.ed.undo_push` before every mutation; transactions wrap N ops in one push |
| Vendoring | `websockets` wheel vendored under `blender_addon/vendor/` (no pip in Blender) |
| Package manager (server) | `uv` (fallback `pip`) |
| Test runner | `pytest` for server; `blender -b -P tests/run.py` for add-on |
| Min Blender | 4.2 LTS |
| Min VS Code | 1.99 (built-in MCP) |

## 14. Security rules (enforced by every task)

1. WS bind = `127.0.0.1` only. Any code that allows non-loopback fails review.
2. Every WS frame validated: must contain `id`, `op`, `auth`. Reject otherwise.
3. Token never logged, never returned in any tool response.
4. `execute_python` MUST pass `safety/validator.py` AST scan: deny `import os|sys|subprocess|socket|ctypes|importlib|builtins`, deny `__*__` attribute access, deny `open()` outside policy roots, deny `eval`, `exec`, `compile`.
5. Every file path arg → `Path(p).resolve()` → must start with one of `policy.allowed_roots`.
6. Rate limit: 50 mutating ops per 10 s per connection (token bucket).
7. Resource caps: mesh polys ≤ `policy.max_polys` (default 1_000_000), render ≤ `policy.max_resolution` (default 4096²).
8. Snapshot `.blend` to `<project>/.blendermcp/snapshots/` before any transaction touching > `policy.snapshot_threshold` (default 5) objects.
9. External strings (object names, file names from disk) wrapped in `<<UNTRUSTED>>…<</UNTRUSTED>>` markers in tool responses.
10. Secrets: no API keys touch the MCP server. Blender token in OS keyring only.

## 19. Glossary

- **MCP** — Model Context Protocol (Anthropic, JSON-RPC over stdio).
- **bpy** — Blender Python API (main-thread only).
- **bmesh** — Blender mesh editing module.
- **Capability** — server-side function executing one Blender op.
- **Tool** — MCP-exposed function the AI can call.
- **Transaction** — group of ops sharing one undo checkpoint.
